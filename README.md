## Project Name:
NETWORK ODYSSEY (or your final title here)

## Description:
NETWORK ODYSSEY is a 2D educational networking game built with Phaser.js. The player navigates a data packet through different network environments, learning core networking concepts such as TCP vs UDP, routing, TTL (Time To Live), encapsulation, encryption, and network congestion.

The game transforms abstract networking principles into interactive gameplay, where each scene represents a different layer or challenge in real-world internet communication.

---

## Category:
Beginner Track — Educational Game / Networking Simulation

---

## How it connects to networking:
This project simulates real networking concepts in a visual and interactive way:

- **Scene 1:** Basic routing between network nodes (IP addresses and routers)
- **Scene 2:** Protocol switching (TCP vs UDP behavior differences)
- **Scene 3:** Encapsulation mechanic (VPN tunneling / packet wrapping)
- **Scene 4:** Network congestion caused by real-world news events (latency spikes / high-frequency traffic)
- **Scene 5:** Final routing decision and delivery to destination node

Players learn how packets move through networks, how congestion affects delivery, and how encapsulation and encryption protect data in transit.

## Demo / How to run:
1. Download or clone the repository
2. Open `index.html` in a browser OR run with Live Server (VS Code)
3. Play through each scene using the “Next” button after completing objectives

---

## Controls:
- Arrow Keys → Move packet (Scene 4)
- Mouse Click → Interact with routers / buttons
- Next Button → Progress between scenes

## Problem tackled:
Networking concepts are often abstract and difficult to visualize. This project solves that by turning packet flow, routing decisions, and network conditions into a playable experience, helping users understand how real-world internet traffic behaves under different conditions.

## What we built:
A multi-scene browser-based game using Phaser.js that simulates packet movement through a dynamic network environment with educational mechanics tied to real networking principles.

---

## How it works:
Each scene represents a different networking layer or concept. The player controls a packet and must make routing decisions, avoid congestion, and successfully deliver data to the destination while learning how real network protocols behave.
